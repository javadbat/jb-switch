<Title>Styling</Title>

`JBSwitch` styling is theme-driven. Use common design tokens for shared product language, component variables for the stable switch API, and CSS parts for official SVG surfaces such as the bar, trigger, and trigger ring.

## Common Design Tokens

Use the core theme docs as the source of truth for shared design-system variables:

- [Theme Colors](../../jb-core/theme/stories/colors.mdx) explains the general color palette variables, default colors, customization approaches, and color generation helpers.
- [Theme Sizes](../../jb-core/theme/stories/sizes.mdx) explains shared size and radius variables.
- [Theme Custom Galleries](https://javadbat.github.io/design-system/?path=/docs/theme-custom-material--docs) gathers the shared custom style recipes across components.

Switch recipes should use those common tokens where possible, then map them into `--jb-switch-*` variables only where the switch needs a component-specific value.

## Styling Model

Use three layers when styling switch examples:

- Common design tokens are documented in [Theme Colors](../../jb-core/theme/stories/colors.mdx) and [Theme Sizes](../../jb-core/theme/stories/sizes.mdx).
- Reusable example theme packs live in [`docs/styles`](https://github.com/javadbat/design-system/tree/main/docs/styles) and can be shared across components.
- Component recipe files live in [`modules/jb-switch/stories/styles`](https://github.com/javadbat/design-system/tree/main/modules/jb-switch/stories/styles).
- Public component variables use the `--jb-switch-*` prefix and are the stable styling API.

Each example uses one shared example theme pack from [`docs/styles`](https://github.com/javadbat/design-system/tree/main/docs/styles) and one switch-specific recipe from [`modules/jb-switch/stories/styles`](https://github.com/javadbat/design-system/tree/main/modules/jb-switch/stories/styles). The style story applies both classes directly to `JBSwitch`.

## Recommended Customization Path

1. Use `--jb-switch-*` variables first for switch internals.
2. Use common theme variables from [Theme Colors](../../jb-core/theme/stories/colors.mdx) and [Theme Sizes](../../jb-core/theme/stories/sizes.mdx) when the style should align with the design system.
3. Put reusable theme-pack values in [`docs/styles`](https://github.com/javadbat/design-system/tree/main/docs/styles) only when they should be shared by multiple component demos.
4. Use `::part(...)` for official exposed SVG surfaces such as `::part(bar)`, `::part(trigger)`, and `::part(trigger-ring)`.
5. Use custom states for state-specific styling, especially `:state(active)`, `:state(inactive)`, `:state(loading)`, and `:state(disabled)`.
6. Add a new component variable only when a needed internal value cannot be expressed with the current public variables or official parts.

## Theme File Pattern

Use [Theme Colors](../../jb-core/theme/stories/colors.mdx) and [Theme Sizes](../../jb-core/theme/stories/sizes.mdx) as the common design-token guides. Theme-pack files should keep reusable demo variables in [`docs/styles`](https://github.com/javadbat/design-system/tree/main/docs/styles), then each component should keep its own style CSS in its story [`styles`](https://github.com/javadbat/design-system/tree/main/modules/jb-switch/stories/styles) folder.

```css
/* https://github.com/javadbat/design-system/blob/main/docs/styles/carbon.css */
.theme-name-style {
  /* shared design-system variables */
}

/* https://github.com/javadbat/design-system/blob/main/modules/jb-switch/stories/styles/style-carbon.css */
.theme-name-switch {
  /* switch-specific variables and official parts */
}
```

This keeps common theme tokens reusable and lets each component page show only the CSS it needs.
